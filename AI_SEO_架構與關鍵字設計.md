# AI SEO 架構與關鍵字設計方案

## 一、AI SEO 是什麼、為什麼要調整

### 1.1 搜尋引擎的變化

- **傳統 SEO**：關鍵字密度、反向連結、標題標籤。
- **AI 時代**：Google SGE（生成式搜尋）、Bing Copilot、答案片段（Featured Snippet）、實體理解（Entity）、語意與意圖理解。

搜尋引擎更在意：

- 內容是否**直接回答查詢意圖**（資訊型、導航型、交易型）。
- 是否具**結構化、可擷取**的資訊（實體、關係、事實）。
- **E-E-A-T**（經驗、專業、權威、可信）與內容品質。
- **使用者體驗**（Core Web Vitals、行動優先、互動與轉換）。

### 1.2 對你網站的意義

月島目前是：**單頁展覽導覽 + 心情測驗 + 訂購入口**。  
AI SEO 要做的是：讓「台南甜點／安南區下午茶／MBTI 甜點／療癒甜點」等查詢，能對應到**清楚的實體、意圖與內容區塊**，並在技術上讓爬蟲與 AI 容易理解。

---

## 二、整體架構：三層對齊

```
Layer 1：關鍵字與意圖設計（查什麼）
    ↓
Layer 2：內容與結構設計（頁面/區塊講什麼、怎麼講）
    ↓
Layer 3：技術與結構化資料（讓爬蟲/AI 怎麼讀）
```

下面依序做：關鍵字設計（二）、內容架構（三）、技術實作（四）。

## 三、關鍵字設計方法

### 3.1 關鍵字分層模型

把關鍵字按「意圖 + 商業價值」分成四層，再對應到網站區塊或未來頁面。

| 層級 | 類型 | 意圖 | 範例 | 對應網站 |
|------|------|------|------|----------|
| 1 | 品牌詞 | 導航 | 月島甜點、Moon Moon、Kiwimu | 首頁、品牌區 |
| 2 | 地域 + 品類 | 發現/比較 | 台南甜點、安南區下午茶、台南蛋糕 | 首頁、地圖、營業資訊 |
| 3 | 需求/情境 | 資訊 + 轉換 | 療癒甜點、MBTI 甜點、預訂取貨、宅配 | 心情測驗、訂購、文案 |
| 4 | 長尾/問句 | 資訊/決策 | 台南安南區甜點店推薦、MBTI 甜點人格 | 文案、FAQ、未來文章 |

| 查詢意圖 | 典型關鍵字 | 對應區塊／元素 | 優先優化 |
|----------|------------|----------------|----------|

## 四、內容架構：對齊關鍵字與 AI 擷取

### 4.1 單頁結構對應（現狀）

- **Hero**：品牌 + 一句價值主張（可埋入「台南」「療癒」「甜點」）。
- **地圖／導覽**：明確寫出「台南安南區」「本原街」「營業時間」。
- **PEAK EXPERIENCE**：對應「心情」「療癒」「MBTI」相關語意。
- **菜單／Archive**：品類詞（提拉米蘇、巴斯克、千層、烤布丁）與簡述。
- **訂購 CTA**：對應「預訂」「取貨」「宅配」意圖。

### 4.2 標題與段落原則（給 AI 與片段用）

- **H1**：一個頁面一個，清楚包含「品牌 + 核心價值或地域」，例：`月島甜點 Moon Moon | 台南安南區療癒系甜點與展覽空間`。
- **H2**：對應大區塊主題，可含一個主關鍵字，例：`台南安南區甜點導覽`、`療癒系甜點與心情體驗`、`預訂取貨與宅配方式`。
- **段落首句**：直接回答「這是誰、在哪、做什麼、為什麼特別」，方便被擷取成摘要或 SGE 答案。

### 4.3 未來可擴充：內容群組（Content Cluster）

若之後有部落格或「更多頁面」：

- **核心頁**：首頁（月島／台南甜點／療癒甜點）。
- **群組 A - 地域**：台南甜點、安南區下午茶、本原街美食。
- **群組 B - 產品**：提拉米蘇、巴斯克、千層、烤布丁、季節商品。
- **群組 C - 體驗**：MBTI 甜點、療癒甜點、心情測驗、展覽。
- **群組 D - 購買**：預訂方式、取貨、宅配、營業時間。

每篇文章鎖定 1 個主關鍵字 + 2～3 個相關詞，並在文內連結回核心頁與菜單／訂購。

## 五、技術實作：讓 AI 與爬蟲更好理解

### 5.1 Meta 與標題（index.html + 動態）

- **Title**：  
  `月島甜點 Moon Moon | 台南安南區療癒系甜點、MBTI 甜點體驗與預訂取貨`  
  （控制在約 60 字元內，含品牌、地域、特色、行動）
- **Description**：  
  150～160 字，包含：誰（月島／Kiwimu）、在哪（台南安南區）、做什麼（療癒甜點、MBTI、展覽）、怎麼買（預訂取貨、宅配）、一句差異化（例如「每一口甜點都是一個故事」）。
- **Keywords**：  
  沿用現有邏輯，但與「關鍵字清單」對齊，補上地域長尾與問句型 2～3 個。

### 5.2 結構化資料（JSON-LD）

在 `index.html` 的 `<head>` 或 `<body>` 開頭加入，讓搜尋引擎與 AI 明確知道「實體類型」與「事實」。

**LocalBusiness（必做）**

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Bakery",
  "name": "月島甜點 Moon Moon",
  "alternateName": "Kiwimu 月島",
  "description": "台南安南區療癒系手作甜點，提供 MBTI 甜點體驗、心情測驗與預訂取貨。",
  "url": "https://moon-island.vercel.app",
  "telephone": "",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "本原街一段97巷168號",
    "addressLocality": "安南區",
    "addressRegion": "台南市",
    "addressCountry": "TW"
  },
  "geo": {
    "@type": "GeoCoordinates",
    "latitude": 23.05,
    "longitude": 120.22
  },
  "openingHoursSpecification": [
    {
      "@type": "OpeningHoursSpecification",
      "dayOfWeek": ["Wednesday", "Thursday", "Friday", "Saturday", "Sunday"],
      "opens": "13:00",
      "closes": "19:00"
    }
  ],
  "priceRange": "$$",
  "servesCuisine": "甜點、蛋糕、咖啡"
}
</script>
```

**WebSite + SearchAction（可選，利於站內搜尋或品牌曝光）**

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "月島甜點 Moon Moon",
  "url": "https://moon-island.vercel.app",
  "description": "台南安南區療癒系甜點與 MBTI 甜點體驗。",
  "publisher": { "@id": "#bakery" }
}
</script>
```

若有「常見問題」區塊，可再加 **FAQPage**，利於 FAQ 片段與 SGE。

### 5.3 語意化 HTML 與無障礙

- 確保只有一個 `<h1>`，其餘區塊用 `<h2>`、`<h3>` 階層。
- 地圖、營業時間、地址用 `<address>` 或明確的區塊與標題。
- 按鈕與連結使用可讀的文案（避免「點這裡」），方便 AI 理解「此連結與什麼意圖有關」。

### 5.4 網址與未來多頁

- 首頁：`https://moon-island.vercel.app/`。
- 若未來有「關於我們／菜單／預訂說明」：  
  `/about`、`/menu`、`/order` 等語意化路徑，並在 JSON-LD 與內部連結中註明。

## 六、AI SEO 工作清單（優先順序）

### 第一階段：立即可做（1～2 天）

1. **Meta 優化**  
   - 改寫 title、description，對齊「關鍵字清單」與「意圖對應表」。  
   - 補齊 og / twitter 的 title、description，與主站一致。

2. **加入 LocalBusiness JSON-LD**  
   - 填正確電話、地址、營業日與時間。  
   - 若有實際經緯度可改寫 `geo`。

3. **H1 / H2 檢查**  
   - 全站一個 H1，內容含「月島／台南／療癒／甜點」等主詞。  
   - 各區塊 H2 對應「導覽／體驗／菜單／訂購」等主題並含一個相關關鍵字。

### 第二階段：內容微調（約 1 週）

4. **Hero 與首屏文案**  
   - 首段 1～2 句內出現「台南」「安南區」「療癒甜點」或「MBTI」等目標詞，且語句自然。

5. **地圖區塊**  
   - 標題或說明含「台南安南區甜點」「本原街」「營業時間」。  
   - 地址以文字形式完整出現一次（利於擷取）。

6. **菜單區**  
   - 區塊標題含「甜點」「蛋糕」或「台南甜點」。  
   - 品項名稱與簡述可自然帶入「提拉米蘇」「巴斯克」「千層」等品類詞。

### 第三階段：擴充與監測（持續）

7. **FAQ 區塊**  
   - 設計 5～10 題：營業時間、預訂方式、取貨、宅配、MBTI 體驗、地址與停車等。  
   - 用 FAQPage JSON-LD 輸出。

8. **關鍵字監測**  
   - 用 GA4 的「自然搜尋」+ Search Console 查詢報告，看「台南甜點」「安南區」「月島」等詞的曝光與點擊。  
   - 依數據微調 title、description 與 H2。

9. **內容群組（若有多頁）**  
   - 依「內容群組」規劃短文或 landing 頁，內部連結回首頁與訂購。

## 八、小結：你這邊可以怎麼做

- **關鍵字設計**：用「品牌／地域／需求／長尾」四層分好，再對應到「意圖 → 區塊」表，之後所有標題與文案都從這張表取材。  
- **AI SEO 架構**：  
  - 內容：H1/H2/首段對齊意圖與關鍵字，未來用內容群組擴充。  
  - 技術：Meta 優化 + LocalBusiness（與必要時 FAQ）JSON-LD + 語意化 HTML。  
- **實作順序**：先改 title/description 與 JSON-LD，再調 H1/H2 與首屏／地圖／菜單文案，最後加 FAQ 與監測。

若你願意，下一步可以直接從「改寫 index.html 的 title / description / keywords」和「加入 LocalBusiness JSON-LD」開始，我可以依你現有文案幫你寫出最終版標籤與結構化資料內容。
